Gaps are variables with floors, not preset tiers.

Three gap roles — edge (head→body, body→claim), body (between sections inside body), row (between cards inside a section). Each ranges from IDEAL = sp-7 (48px) to MIN = sp-2 (8px). On render, JS measures real head/claim heights, computes the body region, and assigns gaps continuously: when there's slack, edge gaps sit at IDEAL and the surplus goes to body gaps so sections stay breathable; when content overflows, edge gaps collapse first to MIN, then body, then row. No discrete tiers — every page picks its own gap values to the pixel.

motivation  ·  surrogate-assisted MOO vs Bayesian optimisation

Why surrogate‑assisted MOO, and not BO / TPE.

Bayesian optimisation already bundles surrogate + acquisition + search into a self-closed loop, so the question is not does the surrogate need updating — BO updates too. The question is which search engine drives the loop, and where each engine collapses.

premise A — evaluation

A single simulation is expensive.

EnergyPlus, Radiance, CFD, FEM: minutes to hours per call. Both BO and SAEA exist because of this.

premise B — search

Engineering search is high‑d, many‑m, batched, mixed.

HPC farms parallelise; performance objectives stack to 4–6; sliders are quantised. BO's acquisition optimiser breaks here.

axis 01  /  batch size

The q wall.

q = candidates per real-sim batch

BO / TPE

Standard EI emits q = 1. q-EHVI O(q!) in the integrand; BoTorch concedes >q ≈ 10 already stretches a single acquisition call. Even q=4 inflates wall-time per step beyond the marginal gain over plain EI batching.

SAEA (this work)

A GA generation is a batch. N=100 evaluated in parallel, infill K=5–20 selected from the live population — no batch-aware acquisition needed. Crossover and mutation give batch diversity for free.

Daulton 2020 q-EHVI, NeurIPS[pdf] BoTorch docs on q-EI scaling cf. InfillStrategy.cs mixed 60/20/20 Wilson 2018 reparam trick[pdf] Balandat 2020 BoTorch arch
axis 02  /  objective count

The m wall.

m = number of objectives

BO / TPE

EHVI closed-form only at m=2; m=3 needs O(N²) box decomp.; m≥4 falls to Monte Carlo, noisy acquisition surface, L-BFGS unstable. Acquisition variance dominates surrogate variance past m=5.

SAEA (this work)

NSGA-III, MOEA/D, RVEA documented up to m=15. Reference vectors and niche-decomposition replace the integral entirely. Reference vectors enforce diversity without explicit hypervolume computation.

Emmerich 2005 EHVI Deb & Jain 2014 NSGA-III[pdf] Cheng 2016 RVEA[pdf] Q. Zhang 2007 MOEA/D
axis 03  /  variable space

The mixed-variable wall.

d quantised, discrete, conditional

BO / TPE

Acquisition optimised by L-BFGS + multi-start — assumes continuity. TPE's KDE degrades past d ≈ 30. Categorical encodings via one-hot blow up the KDE bandwidth selection.

SAEA (this work)

SBX + polynomial mutation handle continuous, integer, categorical in one operator family. Slider quantisation is a post-hoc projection, not a constraint. Operator family is dim-agnostic.

Bergstra 2011 TPE limits Deb 2002 SBX[orig] cf. OptimizerComponentBase.Online.cs:130 Hutter 2011 SMAC mixed types[pdf] Garrido-Merchán 2020 categorical BO
claim
Surrogates solve evaluation scarcity; MOO operators with non-dominated ranking solve search scarcity. BO bundles both jobs into one engine — and that engine breaks at the q, m, and mixed-variable thresholds that engineering optimisation crosses on day one.

How the math works. Available body height = 1080 − (header offset 64 + measured header height) − (measured claim height + footer offset 36 + measured footer height). Try edge=IDEAL, body=IDEAL, row=IDEAL → measure body.scrollHeight. If scrollHeight ≤ available: surplus is split, edge stays IDEAL, body absorbs the surplus (so sections breathe). If scrollHeight > available: shortage is paid down by lowering edge first (toward MIN), then body, then row. If even all-MIN doesn't fit, the overflow class fires.

FRAMEWORK OVERVIEW CHAPTER 2  ·  FRAMEWORK

a complex adaptive system framework for multi-objective generative design

Three branches, one open question.

One main research question splits into three branches — algorithm adaptation, surrogate integration, emergence & dynamics. Each surfaces one structural characteristic of a Complex Adaptive System: adaptation, feedback, emergence. Whether CAS is theoretical core or interpretative lens is the open positioning this thesis must resolve.

main RQ

As optimisation problems grow more complex, can the optimisation procedure itself be designed as an adaptive system — rather than a fixed algorithmic pipeline?

Branch 01  /  algorithm adaptation

Strategy layer.

layer · strategy  ·  trait · adaptation

Sub-RQ 1. Can the optimisation algorithm itself become an adaptive variable, capable of switching or evolving according to system state — rather than being fixed by tool convention (e.g. NSGA-II)?

key questions

  • What indicators reflect problem structure mid-run?
  • When should algorithm switching occur?
  • Is switching beneficial in architectural cases?
planned Year 2  ·  thesis ch. 5

Branch 02  /  surrogate integration

Cognition layer.

layer · cognition  ·  trait · feedback

Sub-RQ 2. If the surrogate participates inside the decision loop instead of being an offline accelerator, how does it alter the system's perception of the problem space — and how should it self-correct against systematic bias?

key questions

  • How to balance prediction and true simulation?
  • How to manage and propagate model uncertainty?
  • Does the surrogate change search dynamics?
in progress Year 1  ·  ch. 3 — see slides 02–06

Branch 03  /  emergence & dynamics

Dynamics layer.

layer · dynamics  ·  trait · emergence

Sub-RQ 3. Is optimisation merely a search for solutions, or does it reveal the structure of the design problem itself — through emergent patterns in Pareto fronts and variable-evolution trajectories?

key questions

  • Can optimisation reveal hidden problem structure?
  • Do different strategies produce different emergent forms?
  • Are trajectory motifs interpretable as system signatures?
planned Year 3  ·  thesis ch. 6
CAS lens

Three traits, one open position.

The three branches each surface one CAS trait — the framework's vocabulary, not (yet) its formal core.

trait · branch 01

Adaptation.

Algorithms adjust to system state — strategy is not fixed at design time.

trait · branch 02

Feedback.

Surrogate ↔ optimiser ↔ environment form a closed loop, not a pipeline.

trait · branch 03

Emergence.

Pareto-front shape and trajectory motifs are not designed — they appear.

open positioning

Should CAS serve as the theoretical core of this thesis, or as an interpretative lens for analysing optimisation behaviour? The choice determines whether the contribution is paradigm-, system-, or algorithm-level.

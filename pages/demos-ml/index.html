DEMONSTRATIONS CHAPTER 2  ·  PRIMER  ·  HOW A SURROGATE WORKS

primer  ·  building intuition for the cognition layer

A surrogate learns from a few expensive points.

Four loops to make machine-learning–assisted optimisation concrete: data appears, a curve fits, the model predicts with uncertainty, and it retrains when fresh ground truth arrives. The dashed grey curve in every panel is the true function the simulator is computing — the surrogate is trying to recover it from samples.

01  ·  expensive ground truth

A few real samples.

Each real simulation is expensive — minutes of EnergyPlus, Radiance, CFD. We can afford only a handful: each crosshair below marks one (x, y) pair we paid for. The dashed curve is the truth we don't know.

02  ·  learn from data

Fit a curve.

The model — Kriging, ELM, neural net, whichever — finds a smooth curve consistent with the samples. It approximates the truth without paying for every x. This is "machine learning" in one sentence.

03  ·  with uncertainty

Predict and σ².

For any new x* the model returns both a prediction ŷ and an uncertainty σ. The teal band is widest where samples are sparse — the model knows what it doesn't know. This is what makes it a surrogate, not just a regression.

04  ·  new ground truth

Retrain when data arrives.

When the surrogate is most uncertain, infill spends real-sim budget there. That new ground truth is added to the training set and the model retrains: the curve adjusts, σ collapses where data just arrived. Online learning, in one loop.

claim
A surrogate is cheap where the simulator is expensive, and honest about its own ignorance — that uncertainty turns infill into a learning loop, not just a sampling routine.
